5.2: Отбор наборов метаданных

Установка элементов предварительно заданных метаданных известна как "наборы метаданных". Примером является набор метаданных Dublin Core. Когда Вы добавляете комплект метаданных к Вашей коллекции, то его элементы становятся доступными для выбора. Вы можете иметь больше,чем один набор. Для предотвращения столкновения одинаковых имён к названию элемента присоединяется короткий идентификатор,который выделяет набор метаданных. Например, элемент Dublin Core "Creator" станет "dc.Creator".Наборы метаданных хранятся в метаданных Библиотечного интерфейса и имеют суффикс ".mds".

Чтобы контролировать набор мета-данных, используемых в коллекции, выберите "Набор Мета-данных" в просмотре "Дизайн".

Список "Присвоенные наборы метаданных" показывает, какие наборы используются в коллекции.

Для того, чтобы использовать другой набор метаданных для уже загруженной коллекции, щелкните "Добавить...". Всплывающее окно отобразит известные GLI наборы метаданных. Чтобы добавить один из них, выберите его из списка и щелкните "Добавить". Если вы определили собственный набор метаданных, вы можете нажать кнопку "Просмотр" для выбора нужного файла.

Для добавления нового набора метаданных щелкните "Новый...". Запустится Редактор наборов метаданных Greenstone, GEMS. В первом всплывающем окне укажите имя набора, пространство имен и описание. Также можно взять за основу один из существующих наборов, в этом случае набор унаследует все элементы из базового. Щелкните OK. Главное окно отобразит слева выбранный набор и некоторые атрибуты набора справа. Если выбрали базой существующий набор, будет отображен один или более элемент. По щелчку на одном из элементов его атрибуты будут отображены справа.

Для добавления нового элемента щелкните правой кнопкой на наборе и выберите "Добавить элемент". Для добавления нового субэлемента щелкните правой кнопкой мыши по элементу и выберите "Добавить субэлемент". Элементы и субэленты можно удалять, выбрав "Удалить (суб)элемент" из контекстного меню.

Замечание: Редактор наборов метаданных Greenstone можно запускать независимо от GLI, выбрав его из главного меню Greenstone, или запустив gens.sh либо gems.bat из папки gli вашей инсталляции Greenstone.

Иногда два набора метаданных могут иметь одно и то же пространство имен, например, Dublin Core и Qualified Dublin Core оба используют пространство имен "dc". Такие наборы нельзя использовать в коллекции вместе. Если вы попытаетесь добавить пространство имен, уже использующееся в вашей коллекции, будет показано предупреждение. Если вы продолжите, существующий набор будет удален, а новый добавлен. Все присвоенные значения метаданных будут перемещены в новый, при условии содержания в нем соответствующих элементов.

Редактирование набора мета-данных выполнено с помощью Greenstone Editor for Metadata Sets (GEMS). Нажмите на кнопку "Редактировать Набор Мета-данных", чтобы получить информацию о GEMS.

Если набор мета-данных вам больше не нужен, выберите его и нажмите на "Удалить Набор Мета-данных". Если вы назначили какие-либо мета-данные к элементам в удаленном наборе, у вас спросят как с ними поступить при следущей загрузке коллекции.
