4.7: «Развертывание» файлов метаданных

Типы файлов метаданных, такие как MARC, CDS/ISIS, BibTex, Refer и Procite можно импортировать в Greenstone, но их метаданные нельзя просмотреть или отредактировать в Интерфейсе библиотекаря. Для просмотра таких метаданных нужно использовать соответствующую им программу.

Развертывание файла базы метаданных делит его на индивидуальные секции с просматриваемыми и редактируемыми метаданными. Процесс необратим: оригинальный файл с метаданными удаляется.

Развертываемые файлы имеют зеленую иконку в Дереве Коллекций. Для разворачивания одной из них щелкните правой кнопкой и выберите "Развернуть базу метаданных". Появится всплывающее окно с опциями развертывания. Первая опция ("плагин") указывает плагин для развертывания. В большинстве случаев, только один плагин будет обрабатывать конкретный тип файлов, но в некоторых случаях,когда разные типы файлов могут иметь одинаковые расширениямогут быть два плагина для этой обработки. Опция "input_encoding" указывает кодировку базы. Опция "metadata_set" указывает, к какому набору метаданных должны быть добавлены новые поля. Если ни одно не указано, вас спросят о каждом из полей: добавить как новый элемент к набору метаданных, объединить с другим элементом, либо игнорировать.

Когда файл разворачивается, создается новый пустой документ для каждой записи, и метаданные из документа присваиваются документу. Они именуются как 000001.nul, 000002.nul и т.д. Если установлена опция «document_field» (имени поля базы данных), то значение этого поля, если оно есть, будет использоваться как имя файла. Процесс развертывания также попытается загрузить файл и использовать его вместо пустого. Опции "document_prefix" и "document_suffix" могут быть использовании для создания правильного URL или пути к файлу из значения поля document_field. Опция "records_per_folder" может быть использована для группирования записей в подпапки. Если база данных очень велика, использование этой опции ускорит последовательное редактирование метаданных.

Развертываемость определяется расширением файла. В некоторых случаях файлы могут быть ошибочно помечены как развертываемые. Например, плагин ProCite обрабатывает файлы с расширением .txt, но большинство .txt файлов – простые текстовые, а не файлы ProCite.
